#01) NAMING CONVENTIONS ================================================================

The eyes are in the 3fac layer, each version (v##) is a different color variance.

The normal eye variants are v01 - v10.  While the inverse eye variants are v11 - v20.

  v01 - Mako Eyes
  v02 - Blue Eyes
  v03 - Green Eyes
  v04 - Red Eyes
  v05 - Brown Eyes
  v06 - Pink Eyes
  v07 - Purple Eyes
  v08 - Steel Eyes
  v09 - Hazel Eyes
  v10 - Honey Eyes
  v11 - Mako Eyes (Inverse)
  v12 - Blue Eyes (Inverse)
  v13 - Green Eyes (Inverse)
  v14 - Red Eyes (Inverse)
  v15 - Brown Eyes (Inverse)
  v16 - Pink Eyes (Inverse)
  v17 - Purple Eyes (Inverse)
  v18 - Steel Eyes (Inverse)
  v19 - Hazel Eyes (Inverse)
  v20 - Honey Eyes (Inverse)

Inverse eyes have the light colored portion of the eyes on top instead of the bottom.

You will also notice there are 2 additional variants listed as v000 and v001.
These are shader variants designed to be recolored in your projects by some
sort of shader or shading method.

An example for Unity would be using one of these shader variants, then changing the
eye colors with the Sprite Renderer's color property.  This would give you a far 
greater variety of eye colors.  The other variants are simply meant for those who 
prefer a more personalized, hand painted touch.

  v000 - Shader Eyes
  v001 - Shader Eyes (Inverse)



#02) USING IN UNITY ================================================================

If you are using the included Unity package to import these graphics into Unity, most
of the work has been done for you.  Obviously, you still have to create your own paper
doll system.

With that having been said, I feel it is important to mention the pivot point on the imported
Unity package.  The pivot points have been set to the position of the character's feet.
If the pivot points in your own project for the bases differ from these, you will want to
set the pivot points of the eyes to match the pivot points of your bases in the sprite editor.
